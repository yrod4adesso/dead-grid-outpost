# US-011 bis US-015 — Acceptance Criteria

## US-011: Chapter Reward Chests
- Reward chests defined per chapter milestone with resources/shards
- `openRewardChest(profile, chapterId, milestoneIndex): DeadGridProfile` — applies rewards
- Chest tracked as opened in `profileProgress.rewardChests`
- Once opened, chest cannot be opened again
- Typecheck passes
- Tests for chest system (at least 6 tests)

## US-012: Daily Objectives System
- Objective definitions with id, title, detail, type, progress tracking
- Objectives generated per session and persist between runs
- `completeObjective(state, objectiveId): DeadGridState` — marks objective complete
- Objective completion state stored in state
- Typecheck passes
- Tests for objectives (at least 6 tests)

## US-013: Tutorial / Onboarding
- Tutorial steps guide new player through core mechanics
- Tutorial progresses through: intro → first mission → first night → rewards
- Tutorial state in state or profile
- Tutorial can be skipped, cannot regress
- Typecheck passes
- Tests for tutorial flow (at least 6 tests)

## US-014: Action Cost Preview
- Preview the cost of actions (recruit, unlock, build, upgrade) before committing
- Preview shows resources needed vs available
- Integrates with building tree prerequisites
- Typecheck passes
- Tests for cost preview (at least 6 tests)

## US-015: Threat Summary
- Threat level calculated from state (buildings, survivors, waves)
- Threat summary shown to player
- Updates dynamically with state changes
- Typecheck passes
- Tests for threat summary (at least 6 tests)
